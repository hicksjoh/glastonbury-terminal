# F10 — 6:30 AM morning push (text only, no voice)

## Status
verified — ready for commit

## Timeline
- Started: 2026-04-21 20:45
- Finished: 2026-04-21 20:52

## Goal
Guarantee Wes gets a push notification at 6:30 AM EDT sharp, without waiting on Claude. The existing `/api/briefing/scheduled` cron was already firing at 6:30 AM EDT, but it generates a full ~4K-token Claude briefing first, so delivery can slip 10-30 seconds. F10 decouples the instant push from the deep briefing.

## What shipped

### `src/app/api/briefing/morning-push/route.ts`
- Vercel cron endpoint, GET + POST. `CRON_SECRET` gating (Bearer or x-api-key).
- Reads live Alpaca account + VIX via `/stable` client — no Claude call, no Supabase briefing row.
- Formats push payload:
  - title: `🌅 Morning snapshot`
  - body: `$EQUITY · overnight +$X (+Y.YY%) · VIX Z`
- Fans out web-push to every `push_subscriptions` row; auto-prunes 410/404 subscriptions.
- Healthchecks.io pings (`briefing-morning-push` slug) at start / success / fail.

### Cron reshuffle (`vercel.json`)
- `30 10 * * 1-5` (6:30 AM EDT) → **NEW** `/api/briefing/morning-push` (lightweight, no Claude)
- `30 13 * * 1-5` (9:30 AM EDT) → MOVED `/api/briefing/scheduled` (deep Claude briefing) from 6:30 AM slot
- Rationale: The 6:30 AM slot is prime real estate for a guaranteed on-time notification. The deep briefing benefits from waiting until after the market opens (9:30 ET) anyway — post-open data makes the briefing more useful.

### `middleware.ts`
- Added `/api/briefing/morning-push` to `PUBLIC_API_ROUTES` so Vercel cron (which doesn't carry a `gt-auth` cookie) can reach it. The route still gates on `CRON_SECRET` internally.

### `e2e/F10-morning-push.spec.ts`

- `@smoke` test (skipped unless `E2E_CRON_SECRET` is set): authed returns `{ ok, subscribers, sent, pruned, payload: { title, body, data } }`
