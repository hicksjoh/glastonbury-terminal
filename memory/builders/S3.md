## Problem
All 6 Vercel cron jobs ran silently — if any failed or stopped firing, nobody would know until stale data caught someone's eye. Classic scheduled-task black hole.

## Solution
Thin helper (`src/lib/healthchecks.ts`) that fire-and-forgets a deadman ping to `hc-ping.com/<key>/<slug>` at the start, end, and failure of each cron. Uses Healthchecks.io's project ping key + auto-provision feature (`?create=1`), so after the first deploy Wes just configures alert channels in the Healthchecks UI — no manual check creation needed.

## Design decisions
- **Fire-and-forget never throws** — a ping failure must never break the cron. All errors swallowed.
- **5-second fetch timeout** via `AbortSignal.timeout()` — prevents a hung hc-ping.com from blocking the cron response.
- **Silent skip if `HEALTHCHECKS_PING_KEY` not set** — safe to deploy before Wes finishes the Healthchecks setup. No throwing, no logging noise.
- **Auto-provision on first ping** (`?create=1`) — Wes doesn't need to pre-create checks in the Healthchecks UI.
- **`start` pings** included so Healthchecks can compute duration + flag long-running runs.
- **Slug naming** — `briefing-scheduled`, `portfolio-snapshot` for the two non-`/cron/*` paths, `cron-<name>` for the four under `/api/cron/`.

## Wes's one-time setup (blocks full observability but not deploy)
1. Sign up at https://healthchecks.io (free tier: 20 checks — we use 6)
2. Create a project "Glastonbury Terminal"
3. Copy the project ping key
4. Set `HEALTHCHECKS_PING_KEY` in Vercel env (Production + Preview)
5. Configure alert channels: email → hicksjoh@gmail.com (recommended), optional Slack/SMS
6. After first successful cron run, each check auto-appears in the project. Set appropriate schedules + grace periods in the UI:
   - `briefing-scheduled`: daily weekday @ 10:30 AM ET, grace 15m
   - `portfolio-snapshot`: daily weekday @ 10:00 PM ET, grace 30m
   - `cron-storm-watch`: daily @ noon UTC, grace 30m
   - `cron-tax-harvest`: weekly Monday midnight, grace 2h
   - `cron-coach-review`: weekly Monday 1 AM, grace 2h
   - `cron-prediction-snapshot`: daily @ 1 PM UTC, grace 30m
