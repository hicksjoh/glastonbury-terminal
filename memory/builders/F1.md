## What shipped

### Streamable-HTTP MCP endpoint at `/api/mcp`
- `src/app/api/mcp/route.ts` — wraps `WebStandardStreamableHTTPServerTransport` from `@modelcontextprotocol/sdk` so the whole JSON-RPC dance runs inside a Next.js App-Router route handler. No Express, no standalone process.
- Stateless transport (`sessionIdGenerator: undefined`) because Vercel serverless invocations don't share memory across requests anyway.
- Bearer-token gated via `MCP_AUTH_TOKEN` env var. Distinct from `gt-auth` so Claude.app can connect without a cookie jar.
- Added `/api/mcp` to `PUBLIC_API_ROUTES` in `middleware.ts` so the route handler itself owns auth.

### Tool catalog (`src/lib/mcp/server.ts`) — 6 tools registered:
- `terminal_get_portfolio` — live Alpaca account + positions + wealth_assets summary + net-worth estimate
- `terminal_get_watchlist` — current watchlist rows
- `terminal_add_to_watchlist` — insert a symbol (uppercased) with optional note
- `terminal_get_latest_briefing` — most recent Keisha briefing row (content + market context + portfolio context)
- `terminal_read_memory` — read cross-agent memory from F14 (single key or list by agent)
- `terminal_write_memory` — upsert into F14 agent_memory with optional TTL

All tool return shapes follow the MCP `CallToolResult` contract (`content: [{ type: 'text', text: ... }]`, `isError?`).

### Dependencies
- `@modelcontextprotocol/sdk@1.29.0` (new)
- `zod@^3` (new — required by McpServer tool schema declarations)

## Bugs fixed along the way
- **`watchlist.added_at` does not exist**. The real schema uses `created_at`. The MCP `terminal_get_watchlist` + `terminal_add_to_watchlist` and the pre-existing `keisha-tools.ts` + `keisha/actions/route.ts` watchlist insert all referenced `added_at`. Fixed to drop the bogus column entirely (Postgres default on `created_at` handles it).
