## Scope pivot
Original scope was "shared primitives: API client wrapper, Widget kit, prompt library" (2 days estimated). After inventorying `src/lib/` and `src/components/` I found the repo already has a production-grade API client (`src/lib/api-client.ts` with rate-limiting + circuit breaker + caching + 20+ API configs) and a full UI primitive kit (StatCard, EmptyState, ErrorBoundary, Skeleton, SparklineChart, PageShell, CommandBar, MarketTickerBar, etc.).

**What was ACTUALLY missing:** Anthropic prompt caching. The 16K-char `KEISHA_SYSTEM_PROMPT` was sent as a plain `system` string on every Claude call in the app — no `cache_control` anywhere in the codebase. That is a ~4K-token write on every call, when it could be a ~400-token cached read.

## What shipped (MVP P1)

### 1. Prompt-caching library
- **NEW:** `src/lib/prompts/index.ts` — `cachedSystem(staticText, dynamicText?)` helper returns the SDK-shaped array of blocks with `cache_control: ephemeral` on the static block. Typed `CachedTextBlock` augmentation handles the v0.24 SDK type lag.
- **NEW:** `src/lib/prompts/keisha-system.ts` — relocated the 16K-char `KEISHA_SYSTEM_PROMPT` out of `src/lib/claude.ts` into its own module.
- **Patched** `src/lib/claude.ts` — re-exports KEISHA_SYSTEM_PROMPT for backward compat; `generateBriefing` and `generateAnalysis` now use `cachedSystem()`.
- **Patched** `src/app/api/keisha/route.ts` and `src/app/api/keisha/stream/route.ts` — split static+dynamic, use `cachedSystem()` on every message.

**Expected savings:** Keisha chat sessions with multiple messages within 5 min now pay ~10% of the system-prompt cost on the 2nd+ messages. Briefing cron always writes the cache so any Keisha interaction within 5 min benefits. At current Sonnet pricing ($3/M input, cached reads ~$0.30/M), the 4K-token prompt drops from ~$0.012 per call to ~$0.0012 per cached call — 10× cheaper.

### 2. FMP /api/v3 migration (tech debt flagged during D1)
- **Patched** `src/app/api/sectors/route.ts`:
  - `fetchQuote` now uses `getQuote` from `fmp-client.ts` (/stable endpoint).
  - `fetchSectorStocksViaScreener` returns `[]` since FMP retired `/api/v3/stock-screener` on this tier and there's no `/stable` equivalent; the route already has a representative-stocks fallback that kicks in cleanly.

### 3. Primitives documentation
- **NEW:** `docs/build-plan/primitives.md` — the "DO NOT REBUILD" inventory. Maps existing modules and components to future feature needs so Wave 1-4 builders grab what exists instead of recreating. Calls out that F15 (⌘K palette) and F16 (ticker strip) are effectively already built.
