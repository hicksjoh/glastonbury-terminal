## Goal
Every agent in the terminal (Keisha, Apollo, future Empire/Franchise/Fed scorers) needs to write and read small structured facts so their observations compound across turns. Today Keisha learns "Wes is stressed about CR3 AR" and Apollo has no idea. Post-F14, `buildSharedContextBlock()` injects that directly into the next agent's system prompt.

## What shipped

### `supabase/migrations/20260421_agent_memory.sql`
- `agent_memory` table: `id, agent_name, key, value jsonb, metadata jsonb, created_at, updated_at, expires_at`
- `UNIQUE (agent_name, key)` so upserts replace cleanly
- Two indexes: `(agent_name, updated_at desc)` for enumeration, `(expires_at)` partial for TTL cleanup
- Trigger that auto-updates `updated_at` on every row UPDATE
- RLS enabled + restrictive deny-all for anon/authenticated; service_role only

### `src/lib/agent-memory.ts`
Tiny API surface — five entry points + one formatter:
- `setMemory(agent, key, value, { ttlMs?, metadata? })` — upsert
- `getMemory(agent, key)` → value or null (auto-skips expired)
- `getRecord(agent, key)` → full row with metadata + timestamps
- `listMemory(agent, limit=50)` — newest-first, expired skipped
- `deleteMemory(agent, key)` — idempotent
- `cleanExpired()` — for weekly cron
- `buildSharedContextBlock(maxChars=2000)` — formats shared memory for prompt injection, caps length so it stays inside one cache breakpoint

Convention: `agent_name = 'shared'` for memory anyone can read. Per-agent scopes keep private scratch out of other agents' context.

### `src/lib/__tests__/agent-memory.test.ts`
10 vitest cases with a flexible chain-builder Supabase mock. Covers:
- upsert shape
- get/getRecord round-trip
- expired-row filtering (get + list)
- delete success/failure
- cleanExpired count propagation
- unconfigured-env graceful no-op (all six functions)
- `buildSharedContextBlock` formatting + empty-list empty-string

### Integration hooks (intentionally left for callers)
The library is deliberately unopinionated. Wiring:
- Keisha briefing or streaming route can prepend `await buildSharedContextBlock()` to the dynamicContext block that already flows through `cachedSystem()` from P1.
- Any agent can call `setMemory('shared', 'user_mood', { label: 'stressed', source: 'keisha-chat-msg-123' })` when it detects signal.
- A future cron (Wave 4 or sooner) runs `cleanExpired()` weekly.
